number input and variable

a

Input of type number, thus up and down arrows increment and decrement.
keydown.i is bound to increment, keyup.r to reset.

b

Input of default type text.
Up and down arrow keys are bound to increment and decrement.

c

Two way binding with [(ngModel)].

Text input makes the variable disappear (rather than showing NaN or 0).

d

The arrow keys are bound to moving the focus.
arrowUp also increments (default behaviour), but arrowDown is kept from decrementing by preventDefault.

The variable (seen right to the input) changes only when the input is changed by typing or (mouse) clicking on the arrows.
It does not on pressing arrowUp, although the number in the input increases.

e

Two way binding with [ngModel] and (ngModelChange).

Ctrl+arrowUp will move the focus. Again, this increases the number in the input, but not the variable.
